# Problem
Initial implementation of smart contracts was needed to handle token minting, saving plan administration, deposit opening, and interest payments.

# Reason
Core business logic requires precise mathematical formulas for interest calculations and strict separation of roles between `SavingCore` and `VaultManager`.

# Solution
1. Implemented `MockUSDC.sol` with 6 decimals for testing environment.
2. Implemented `VaultManager.sol` with `fundVault()`, `withdrawVault()`, and `payInterest()` restricted to `SavingCore`.
3. Implemented `SavingCore.sol` with `createPlan()`, `openDeposit()`, and linear interest formula: `(principal * aprBps * tenorSeconds) / (365 days * 10000)`.

# Decision
When a user opens a deposit, `SavingCore` transfers ERC20 tokens into `SavingCore` and mints an ERC721 certificate NFT to the depositor. `VaultManager` holds separate funds specifically allocated for interest.
