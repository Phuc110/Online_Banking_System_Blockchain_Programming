## Startup Workflow

Every time you start working on this project, follow these steps in order.

1. Read PLAN.md
2. Read CONTEXT.md
3. Read ARCHITECTURE.md
4. Read CHANGELOG.md
5. Continue unfinished tasks.

Never ask what to do next if PLAN.md already contains unfinished tasks.

## Coding Rules

Always:

• Write clean and maintainable code.

• Follow SOLID principles.

• Follow OpenZeppelin best practices.

• Keep functions small.

• Prefer readability over cleverness.

• Never duplicate logic.

• Never remove working code unless requested.
