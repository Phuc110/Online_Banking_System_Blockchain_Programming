# Documentation Skill

## Goal
Maintain comprehensive, production-quality documentation for the project following the startup and completion workflows.

## Mandatory Documentation Files
- **PLAN.md**: Active and completed milestones, feature tasks, and bug fixes.
- **CONTEXT.md**: Domain business logic, roles, entity rules, and saving plan mechanics.
- **ARCHITECTURE.md**: System diagrams, contract interactions, security invariants, and design patterns.
- **CHANGELOG.md**: Version history tracking bug fixes, refactoring, and new feature additions.
- **conversations/YYYY-MM-DD.md**: Concise decision logs recording problem, root cause, solution, and decision.

## Workflow Rules
- Update `PLAN.md` and `CHANGELOG.md` upon completing any task.
- Update `CONTEXT.md` when business logic or rules change.
- Update `ARCHITECTURE.md` when contract interfaces or interactions change.
