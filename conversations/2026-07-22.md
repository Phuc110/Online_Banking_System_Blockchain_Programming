# Problem
1. `npx hardhat test` failed because `MockUSDC.deploy()` in `test/MockUSDC.test.js` missed the required `initialOwner` parameter expected by OpenZeppelin v5 contract constructor.
2. In `SavingCore.sol`, `renewDeposit()` lacked `require(block.timestamp >= oldDep.maturityAt, "Not matured")`, enabling users to call renewal immediately after opening a deposit and repeatedly drain interest from `VaultManager`.
3. In `SavingCore.sol`, `autoRenewDeposit()` burned the old deposit NFT and minted a new deposit NFT but forgot to calculate and send the accrued interest to the owner.
4. Raw ERC20 `.transfer()` and `.transferFrom()` calls were vulnerable to non-standard ERC20 tokens.
5. Missing project governance docs (`PLAN.md`, `CONTEXT.md`, `ARCHITECTURE.md`, `CHANGELOG.md`), incomplete test suites, and unformatted `.agents/skills` configs.

# Reason
Incomplete contract security validation, missing OpenZeppelin v5 parameters in test deployments, missing unit tests for `SavingCore` and `VaultManager`, and unstandardized agent skill metadata.

# Solution
1. Updated `MockUSDC.deploy(owner.address)` in test scripts.
2. Added `require(block.timestamp >= oldDep.maturityAt, "Not matured");` to `renewDeposit` in `SavingCore.sol`.
3. Calculated accrued interest and added `vault.payInterest(owner, interest)` call in `autoRenewDeposit` in `SavingCore.sol`.
4. Applied OpenZeppelin `SafeERC20` (`safeTransfer`, `safeTransferFrom`) across `SavingCore.sol` and `VaultManager.sol`.
5. Created 66 comprehensive unit tests across `test/VaultManager.test.js` and `test/SavingCore.test.js`.
6. Created root and `.agents` documentation files (`PLAN.md`, `CONTEXT.md`, `ARCHITECTURE.md`, `CHANGELOG.md`, `README.md`) and refactored all skills in `.agents/skills/` with YAML frontmatter.

# Decision
All 66 tests pass. The smart contracts are now secured against premature renewal attacks, auto-renewal interest loss, and non-standard ERC20 failures. All workspace documentation and skill configurations are fully aligned with `AGENTS.md` guidelines.
